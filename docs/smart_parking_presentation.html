Smart Parking System — End-to-End

Reduce congestion, save time and emissions by combining low-cost IoT sensors with predictive machine learning.

  • Problem: ~30% of traffic is from parking search.
  • Solution: Real-time sensor status + ML forecasts for future availability.
  • Outcome: Faster parking, better utilization, lower emissions.
Prepared: ParkSpot Team
Duration: ~12 slides

System Architecture

Hardware → Microcontroller → WiFi Module → Flask Server → CSV / DB → ML Model → Dashboard
graph LR IR[IR Sensor] -->|Signal| MEGA[Arduino Mega] MEGA -->|Serial Data| ESP[ESP8266] ESP -->|HTTP POST| SERVER[Flask Server]

Data Pipeline & ML

Data ingest, hourly aggregation, features: hour (0-23), day_of_week (0-6). Model: MultiOutput RandomForestRegressor to predict probability of each parking slot being free.

  • Preprocess: timestamp → hour, dayofweek, pivot to slots
  • Train/Test: typical 80/20 split
  • Model: RandomForest in sklearn, wrapped with MultiOutputRegressor

Features & Metrics

Features
  • Hour of day (continuous)
  • Day of week (categorical encoded 0-6)
  • Historical occupancy statistics (rolling mean)
Evaluation
  • Use ROC-AUC per slot where applicable
  • Precision/Recall for binary availability predictions
  • Business metric: % reduction in search time (A/B test)

Deployment & API

Flask app exposing endpoints for sensor POSTs and a web UI for predictions. Minimal example endpoints:

POST /api/sensor  → logs CSV
GET /predictions?hour=14.5&day=2 → returns slot probabilities

Consider adding auth, rate limits, and batching at the API gateway for production.

UX & Dashboard

Design the dashboard for quick scanability:

  • Heatmap of occupancy by hour/day
  • Recommender: nearest available slot
  • Export CSV / JSON for reporting

Hardware Implementation Tips

  • Place IR sensors angled to avoid false positives from pedestrians.
  • Use shielded cabling for long runs; debounce sensor readings.
  • Periodically calibrate sensors and monitor drift.

Production Checklist

  1. CI for model retraining and validation
  2. Monitoring (latency, failed posts, model drift)
  3. Backups for CSV → move to time-series DB (InfluxDB / TimescaleDB)
  4. Secure endpoints and device authentication

Business Case & ROI

Estimate reduced congestion and driver time savings. Highlight payback from optimized parking operations and improved customer experience.

  • Reduced search time → less fuel, better throughput
  • Data-driven pricing & allocation

Next Steps

  • Run pilot on one lot (2-4 weeks)
  • Collect more data, retrain weekly
  • Integrate with parking payment / wayfinding
